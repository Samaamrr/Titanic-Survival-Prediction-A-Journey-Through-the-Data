NOTES:

  1. For imbalanced datasets, accuracy can be misleading. In such cases, F1 Score, Precision, Recall, or ROC-AUC are preferred.
  2. If false positives and false negatives have different costs, use Precision and Recall.
  3. For a general view of model performance, ROC-AUC provides insight into how well the model differentiates between classes across various thresholds.
  4. Use F1 Score when you need a balance between precision and recall, especially with imbalanced data.

Key Points:

  1. Recall (Sensitivity):

What It Measures: How well the model finds all the positive cases. When Important: When missing a positive case is bad (e.g., diagnosing a disease). You want to catch every possible positive, even if it means some false positives.

  1. Precision:

What It Measures: How accurate the model’s positive predictions are. When Important: When false positives are costly or problematic (e.g., spam filtering). You want to make sure that the positives predicted by the model are actually positive.

Simple Guidelines:

  1. If Missing a Positive Case is Costly (e.g., Disease Detection): Focus on Recall. You want to catch every positive case, even if it means getting some false alarms.

  2. If False Positives are Costly (e.g., Spam Detection): Focus on Precision. You want to make sure that what the model predicts as positive is actually positive.
